# Stage 2 Part 1: Refined Source Code

Stage 2 Part 1 evaluates the source code for your refined Stage 1 workflow or agent.

The final proposal PDF is submitted in Stage 2 Part 2. Part 1 focuses only on source code.

## Due Date

Stage 2 Part 1 is due Friday, June 12, 2026, 11:59 PM Pacific Time.

Late submissions are accepted until Sunday, June 14, 2026, 11:59 PM Pacific Time with a 20% penalty.

## Required Deliverables

Submit:

- source code for your refined agent or workflow

You do not need to submit `workflow_usage.md`, run logs, screenshots, transcripts, demo evidence, or `AI_USAGE.md` for Part 1.

## Source Code Expectations

Your source code should make it possible to review the refined workflow or agent. It should show meaningful improvement beyond Stage 1 and connect clearly to proposal-writing needs such as problem framing, novelty, method, evaluation, feasibility, source handling, or revision.

Use the Stage 2 Part 1 section of [grading_rubric.md](grading_rubric.md) for detailed scoring expectations.

## Important Rule

Part 1 credit is based on submitted source code. Part 2 credit is based on the final proposal quality.

## Stage 2 Part 1 Checklist

- Source code for the refined agent or workflow is included.
- Source code shows a meaningful refinement beyond Stage 1.
- Source code is organized and reviewable.
- Source code connects to proposal-writing requirements.
