# Stage 2 Part 2: Final Proposal

Stage 2 Part 2 evaluates the final proposal itself. The proposal should read like a research proposal, not a short course implementation report.

## Due Date

Stage 2 Part 2 is due Friday, June 12, 2026, 11:59 PM Pacific Time.

Late submissions are accepted until Sunday, June 14, 2026, 11:59 PM Pacific Time with a 20% penalty.

## Required Deliverables

Submit:

- `proposal.pdf`
- `proposal.tex` or equivalent proposal source
- references or source notes
- figure or diagram source if applicable

## What The Proposal Should Show

The final proposal should:

- follow the required format and page limit;
- include a clear figure, diagram, workflow chart, or architecture sketch;
- explain the problem, motivation, and target context;
- explain novelty and relation to prior work;
- describe the method or agent workflow concretely;
- include a realistic evaluation plan;
- explain milestones, risks, resources, and feasibility;
- mark unsupported claims as assumptions or connect them to sources.

## Important Rule

Part 2 is graded for proposal quality. Part 1 is graded from the submitted source code for the refined agent or workflow.

## Stage 2 Part 2 Checklist

- Final file is submitted as `proposal.pdf`.
- Proposal source is included.
- Figure is readable and referenced in the text.
- Novelty is compared against relevant prior work, examples, tools, or workflows.
- Evaluation plan has concrete tests or metrics.
- References, source notes, or assumptions are included.
